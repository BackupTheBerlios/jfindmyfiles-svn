A correcta avaliação das duas plataformas deveria ser feita recorrendo à 
implementação de um mesmo sistema de forma a ser possível comparar o impacto 
que as diferentes arquitecturas, funcionalidades e características têm no 
desenvolvimento, no entanto, o curto tempo disponível para desenvolver todo o 
sistema, invalida essa metodologia de avalição por parte da equipa de 
desenvolvimento. Assim recorreu-se ao estudo feito por outro profissional 
da área, que desenvolveu uma mesma aplicação nas duas plataformas e publicou o 
seu estudo. [TEXTO INCORRECTO, REESCREVER!]

Na conclusão da análise determina-se que ambas as plataformas possuem 
caracteristicas similares, sendo os pormenores de implementação e arquitectura 
pouco relavantes para a escolha, isto é, através da comparação das capacidades 
técnicas das duas plataformas o resultado é um empate.

Sendo assim, e tendo em conta os requisitos da aplicação efectuou-se uma

Outros pontos de relevo.

- Disponibilidade de documentação oficial, existência de exemplos práticos.

Ambas as plataformas possuem uma sólida documentação oficial com bastantes 
exemplos e caso de uso sobe o desenvolvimento de aplicações ricas. A documentação
existente é actualizada e facilmente acessível.

Neste ponto a plataforma Netbeans leva um ligeira vantagem na medida em que 
providencia pequenos tutoriais onde se desenvolvem aplicações completas sobre a 
plataforma.
Existe também um módulo para o IDE Netbeans que permite ter acesso directo, dentro 
do IDE, à documentação da API em modo offline.

- Comunidade

As comunidades das duas plataformas são também bastante vastas, com listas de 
discussão, fóruns, e wikis onde as dúvidas poderão ser respondidas rápidademente.

- IDEs que se podem usar

Infelizmente a escolha da plataforma irá limitar o tipo de IDEs que se poderão 
usar, nesse sentido, a escolha da plataforma Eclipse RCP irá condicionar à 
utilização do IDE Eclipse, do mesmo modo que a escolha da plataforma Netbeans 
irá condicionar à utilização do IDE Netbeans.

Nenhum outro IDE poderá ser usado se o objectivo é tirar partido das 
funcionalidades e vantagens que o uso de uma plataforma deste género oferece.
Embora seja teóricamente possível programar usando outros IDEs e outras 
ferramentas na prática isso resultará na perda de todas as facilidades existentes, 
como assistentes e geração automática de código e aumentará a carga de trabalho 
aos programadores.

- Sistema gráfico

A plataforma Netbeans é desenvolvida e usar o sistema gráfico Swing, este sistema 
está disponível com todas as edições da plataforma Java e é o standard nesta 
tecnologia. Questões de velocidade que assombravam este sistema estão actualmente
ultrapassadas sendo este sistema cumpre o requisito de portabilidade imposto.

A plataforma Eclipse RCP usa o sistema gráfico SWT. Apesar de ser anunciado como 
um sistema nativo, rápido e multiplataforma, alguns problemas foram identidicados 
que contradizem esse anúncio.
A tecnlogia SWT não é enviada com a plataforma Java, não fazendo parte desta, e 
por esse mesmo motivo não está tão rapidamente disponível em novos sistemas.
A utilização desta tecnologia irá também expor os programadores a bugs específicos 
da plataforma aumentando assim a complexidade de desenvolvimento.

- Portabilidade da plataforma

A plataforma Netbeans não apresenta problemas de portabilidade além dos existentes 
na tecnologia Java.

A plataforma Eclipse RCP introduz alguns problemas específicos ao sistema gráfico 
que usa, como indicado no ponto anterior e posteriormente.

- Conhecimento que os elementos do grupo possuem sobre as plataformas e sistema gráfico

Ambas as plataformas são desconhecidas para os programadores, embora estes tenham 
já alguma experiência com o sistema gráfico da plataforma Netbeans, Swing, e com 
IDE Netbeans.

O sistema gráfico da plataforma Eclipse RCP, SWT, é completamente desconhecido 
para os programadores. O uso do IDE Eclipse também não é comum na equipa de 
desenvolvimento.

- Outros pontos de relevo.

Irá ser necessária a integração de bibliotecas que foram desenvolvidas usando 
Swing. No caso da escolha da plataforma Eclipse RCP essa integração ficará limitada 
a sistema Linux e Windows devido a alguns bugs existentes no sistema SWT que 
impendem essa integração de funcionar correctamente no sistemas operativos 
Mac OS X.

- Conclusão
Tendo em conta os pontos acima indicados decidu-se a escolha da plataforma 
Netbeans. Esta plataforma vai de encontro a todos os requisitos apresentados e 
é a plataforma que introduz menos problemas à equipa de desenvolvimento
