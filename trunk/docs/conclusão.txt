Conclusões:


Na implementação desta aplicação foi usada a metodologia XP permitindo uma análise menos extensa.
Foi também usada a linguagem JAVA a qual é portável permitindo assim a compilação da aplicação uma vez
apenas para que essa possa ser executada em qualquer plataforma que possua a máquina virtual Java. 
Não existe assim a necessidade de produzir uma versão compilada para cada SO em que se deseje executar a aplicação.

Pode-se concluir que a realização da aplicação permitiu criar um sistema de gestão de 
ficheiros de fácil utilização, catalogação, multiplataforma, com licença Open Source. 
A portabilidade foi um aspecto muito importante a ter em conta visto que um utilizador pode possuir vários
SOs e deste modo não precisa de andar com várias aplicações com as mesmas utilidades que 
a JFindMyFiles. Assim usa só esta aplicação para qualquer SO e não precisa de se preocupar
com o facto de a aplicação não importar o ficheiro que tinha sido exportado noutro computador.

As funcionalidades definidas inicialmente foram quase todas implementadas a excepção das iterações 8 e 9 deixando
para desenvolvimento futuro. De um modo geral, a aplicação possui funcionalidades que existem já nos outros sistemas 
(aplicações/software) as quais são úteis na actualidade. Possui a funcionalidade de ser multiplataforma, a qual 
ainda não existe nos outros sistemas existentes, sendo uma vantagem.
